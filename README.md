# Бенчмарк производительности структур данных и алгоритмов

## Описание проекта

Данный проект представляет собой комплексное исследование производительности различных структур данных и алгоритмов, реализованных в рамках заданий 2, 3 и 5:

- **Задание 2**: Хеш-таблица с разрешением коллизий методом цепочек
- **Задание 3**: Неблокирующая (lock-free) хеш-таблица
- **Задание 5**: Красно-чёрное дерево и ассоциативный массив на его основе
- **Дополнительно**: Сравнение алгоритмов сортировки (быстрая сортировка vs сортировка вставками)

Проект выполняет замеры времени выполнения операций вставки, поиска и удаления для различных структур данных, а также сравнивает производительность алгоритмов сортировки.

## Технические особенности

### 1. Смешанный C/C++ код

Проект использует как C, так и C++ код:
- **C**: Хеш-таблица из задания 2
- **C++**: Неблокирующая хеш-таблица, КЧД, бенчмарк (используют классы, STL, атомарные операции)

Для связи между языками используются:
- `extern "C"` — для вызова C-функций из C++
- Обёрточные функции — для вызова C++-функций из C

```cpp
#ifdef __cplusplus
extern "C" {
#endif
    void benchmark_lockfree_hash_task3_int(int n);
    void benchmark_rbtree_task5_int(int n);
#ifdef __cplusplus
}
#endif
```

### 2. Высокоточное измерение времени
Для измерения времени используется QueryPerformanceCounter() на Windows:

```c
LARGE_INTEGER frequency, start, end;
QueryPerformanceFrequency(&frequency);
QueryPerformanceCounter(&start);
// ... выполняем операцию ...
QueryPerformanceCounter(&end);
double time_ms = (double)(end.QuadPart - start.QuadPart) / frequency.QuadPart * 1000.0;
```

Преимущества:
- Микросекундная точность
- Независимость от загрузки системы
- Стабильные результаты

Альтернативы:
- clock() — менее точный
- std::chrono — для C++ части

### 3. Глобальные переменные для обмена данными
Для передачи результатов между C и C++ частями используются глобальные переменные:

```cpp
// В benchmark.cpp
BenchmarkResult g_hash_result = {"Хеш-таблица", 0, 0, 0, 0, 0};
BenchmarkResult g_lockfree_result = {"Неблокирующая HT", 0, 0, 0, 0, 0};
BenchmarkResult g_rbtree_result = {"Красно-чёрное дерево", 0, 0, 0, 0, 0};

// В benchmark.h — объявление
extern BenchmarkResult g_hash_result;
extern BenchmarkResult g_lockfree_result;
extern BenchmarkResult g_rbtree_result;
```

## Структуры данных

### 1. Хеш-таблица с цепочками (задание 2)

Характеристики:
- Разрешение коллизий: метод цепочек
- Ёмкость: степень двойки (оптимизация & (capacity - 1))
- Порог перехеширования: 75%
- Хеш-функция: djb2 с дополнительным перемешиванием

Сложность:
- Вставка: O(1) средняя, O(n) худшая
- Поиск: O(1) средняя, O(n) худшая
- Удаление: O(1) средняя, O(n) худшая

### 2. Неблокирующая хеш-таблица (задание 3)

Характеристики:
- Фиксированный размер: 1024 бакета
- Разрешение коллизий: метод цепочек
- Логическое удаление (флаг deleted)
- CAS-операции для потокобезопасности
- Защита от ABA-проблемы (TaggedPtr)

Особенности:
- Вставка: CAS-цикл
- Поиск: wait-free (без циклов)
- Удаление: логическое (установка флага)
- Периодическая физическая очистка

### 3. Красно-чёрное дерево (задание 5)

Характеристики:
- Самобалансирующееся BST
- Фиктивный лист nil для упрощения
- Свойства: корень чёрный, красные узлы не имеют красных детей, одинаковая чёрная высота

Сложность:
- Вставка: O(log n)
- Поиск: O(log n)
- Удаление: O(log n)

### 4. Алгоритмы сортировки

#### 1. Быстрая сортировка (Quick Sort)

Сложность: O(n log n) средняя, O(n²) худшая

Принцип: разделяй и властвуй (разбиение относительно опорного элемента)

#### 2. Сортировка вставками (Insertion Sort)

Сложность: O(n²) средняя и худшая

Принцип: построение отсортированной последовательности путём вставки элементов
